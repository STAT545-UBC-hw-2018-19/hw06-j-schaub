### String basics

**1. In code that doesn’t use stringr, you’ll often see `paste()` and `paste0()`. What’s the difference between the two functions? What stringr function are they equivalent to? How do the functions differ in their handling of `NA`?**

The difference is the default for `sep`. `paste()` has a default value `sep = " "` if nothing is specified. `paste0()` has a default value `sep = ""` if nothing is specified.

``` r
paste("Je", "ssi", "ca")
```

    ## [1] "Je ssi ca"

``` r
paste0("Je", "ssi", "ca")
```

    ## [1] "Jessica"

`paste()` and `paste0()` don't handle NA's well.

``` r
a <- c("a", NA, NA)
b <- c("x", "y", NA)

paste(a, b)
```

    ## [1] "a x"   "NA y"  "NA NA"

``` r
paste0(a, b)
```

    ## [1] "ax"   "NAy"  "NANA"

This default output of "NAy" is not a result I can imagine being useful in many situations. However, a more intuitive way to deal with NA's in paste does not really exist yet.

**2. In your own words, describe the difference between the `sep` and `collapse` arguments to `str_c()`.**

`str_c()` can concatenate multiple vectors together. `sep` controls how the elements of the different vectors are separated once they are joined. `collapse` controls how the final concatenated vectors will be collapsed down into one final string. Without `collapse`, the output will be the same length as the vectors that were input. With `collapse`, the length will be 1.

``` r
#Build vectors for example
vec1 <- c("x", "y", "z")
vec2 <- c("a", "b", "c")

#Show with sep
#output is the same length as the longest vector
str_c(vec1, vec2, sep = "&")
```

    ## [1] "x&a" "y&b" "z&c"

``` r
#Show with sep & collapse
#output is one string
str_c(vec1, vec2, sep = "&", collapse = "-")
```

    ## [1] "x&a-y&b-z&c"

Finally, we can make the data untidy for our analysis. There are two types of data represented in this data set:

-   Candy
-   Questions about other things

So let's split the data frame into two data frames: `candy` & `questions`. Metadata for ID, age, timestamp, and trick\_or\_treating will be preserved between both data frames.

Let's build the `candy` data frame. Since all candy values have square brackets around them, we can search for these. However, square brackets are found in some questions and these questions all start with 'Please'. We will use this to our advantage.

``` r
# All columns with [, with the exception of questions that contain [ (these begin with 'Please')
candy <- candy_all %>% 
  select(id,
         age,
         time,
         trick_or_treating,
         contains('['),           #candy names, all contain []
         -contains('Please'))     #without (-) questions that contain []

#Check our work
candy %>% 
  names() %>% 
  kable()
```

``` r
#isolate candy names, then remove brackets
# saved to vector 'names'

  select(contains('['),

  names() %>%

  str_replace(pattern = "\\]", replacement = "")
```
